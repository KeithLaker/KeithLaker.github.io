New paradigm for case-sensitivity and collation on Oracle Database 12c Release 2
Moscone West - Room 3008 Sergiusz Wolicki, Oracle

Historically, the way an Oracle Database compares and sorts character data is controlled in a pretty coarse way by the parameters NLS_SORT and NLS_COMP. This session will present a new feature of Oracle Database 12c Release 2 (12.2) that introduces a declarative method of associating collation with particular data. Using the new mechanism, called Data-Bound Collation (DBC), the user can precisely control how any given character data is compared depending just on the source of the data. For example, content of given columns may be declared as case-insensitive. The new functionality is based on the ISO/IEC SQL standard, which simplifies application porting to Oracle Database from other RDBMS systems that support similar functionality.

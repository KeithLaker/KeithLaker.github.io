Revolutionize Analytics with Oracle Database In-Memory – Latest and Futures
MOSCONE SOUTH 102D

With Oracle Database In-Memory’s industry-first “dual format” architecture, users can transparently accelerate analytics by orders of magnitude and get immediate answers to questions that previously took hours or longer. Memory Without Bounds-Policy Based Automation of In-Memory Column Store Content
MOSCONE SOUTH 103E

Automatic Data Optimization (ADO) has been extended to support Oracle Database In-Memory. ADO policies can be used to automatically manage the contents of the Oracle Database In-Memory column store. Learn how you can add the In-Memory column store to your Information Lifecycle Management (ILM) strategy by defining both heat map-based and custom policies to control the population and eviction of objects, as well as the recompression of objects to a higher level within the Oracle Database In-Memory column store. This session will also cover how to combine Oracle Database In-Memory, database compression, and storage tiering policies to provide a comprehensive ILM solution to automatically manage and optimize your Oracle Database storage.

Database Actions - Data Modeler

Data Modeler is a built-in graphical tool that enhances productivity and simplifies data modeling tasks. Using Data Modeler users can create, browse and edit, logical, relational, physical, multi-dimensional, and data type models.

The Data Modeler tools provides forward and reverse engineering capabilities and supports collaborative development through integrated source code control. The Data Modeler can be used in both traditional and in Cloud environments.

Data Engineering With Data Studio

Data Studio is bundled with each Autonomous Database instance and encompasses several key data engineering tools:

Data Load:
This tool allows users to load data from various sources into their Autonomous Database. Supported sources include local files, remote databases (via database links), and cloud storage buckets in Oracle Cloud Infrastructure (OCI), Amazon S3, Azure Blob Storage/Data Lake Storage, and Google Cloud Storage. It supports numerous file formats like CSV, XLS, JSON, Parquet, GeoJSON, and more. It also facilitates creating Live Feeds for repeated data loads. Data Load integrates with OCI Language Service capabilities to perform sentiment analysis, key phrase extraction, language detection, and text translation on data during the loading process.

Catalog:
The Catalog tool enables users to understand data dependencies and the impact of changes. It supports multi-catalog capabilities, allowing discovery and use of data across a wide range of external systems, including other Autonomous Databases, on-premises databases, cloud storage, shared data, and other data catalogs like OCI Data Catalog and AWS Glue. Users can browse, search, and manage entities and register cloud links.

Data Insights:
This page helps users discover anomalies, outliers, and hidden patterns within their data.

Data Analysis:
This tool focuses on analysing data and creating Analytic Views and reports. It allows creating Analytic Views with multidimensional metadata on fact tables, defining hierarchies and measures. Users can analyse and visualize data, inspect data with pivot tables, find errors, generate insights, and perform hierarchical aggregation and drill-down. Natural Language Query is supported, allowing users to ask questions in plain language instead of writing SQL. It also supports Vector Search. Analytic Views can be exported to tools like Tableau and Oracle Analytics Cloud (OAC) for visualization, and some benefits can be gained by mapping Power BI to transparency views.

Data Transforms:
Providing a graphical user interface, this tool allows users to design data transformations for data integration without writing code. It supports creating data loads, data flows (defining movement and transformation), and workflows (sequencing data flows). Data Transforms is available as part of the Data Studio suite and also as a separate Marketplace listing (Data Integrator: Web Edition). It requires ADMIN or DATA_TRANSFORM_USER role. It can connect to various technologies and supports building data warehouses from sources like NetSuite.

Table AI Assist:
This AI-powered tool helps users augment and correct data in local tables using natural language prompts. Users can ask the tool to add, rename, or remove columns using natural language, and it generates the corresponding SQL expressions. It uses a Large Language Model (LLM) configured via the Data Studio settings and an AI profile.

Generative AI

ADB integrates generative AI directly into its architecture without requiring data movement to separate services. This contrasts with some other data warehouses that might require integrating multiple additional services for generative AI or machine learning functionalities.

Here's how ADB uses generative AI:

Select AI for In-Database Generative AI:
Autonomous Database features Select AI, which provides automated and integrated generative AI. It uses in-database Large Language Models (LLMs) and an automated, in-database vector store.

Select AI enables contextual conversations in natural language, allowing users to leverage generative AI without needing existing AI expertise or moving data out of the database. Unlike copilots or advisors that might be separate services, Select AI is immediately available directly for any application that can connect to the database.

ADB allows integration with an LLM of choice from various popular providers across preferred clouds, designed to easily integrate with new LLMs as they become available.

Natural Language Interaction and Querying:
Select AI enables users to "speak human" to ask questions about their business instead of writing SQL queries. The Data Analysis tool within Data Studio supports Natural Language Query, allowing users to query the database using plain language. APEX ChatDB accelerates the development of business-focused, GenAI chat-driven applications.

AI-Powered Data Catalog and Marketplace:
In the Data Marketplace tool, AI capabilities are used to auto-fill dataset descriptions and tags for shared listings.

AI Explain:
Autonomous Database supports generating Natural Language explanations from SQL queries using AI Explain, helping users understand existing SQL.

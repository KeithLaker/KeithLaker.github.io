Autonomous Database - Costing Model for a Modern Data Warehouse


Oracle Autonomous Database

Example Of A Costing Model For A Modern Data Warehouse

For the sake of simplicity, the examples below assume usage in set blocks of time (either hours or minutes) Actual billing with Oracle Cloud is calculated using per-second billing. Therefore, it's highly likely that actual running costs could be considerbly lower than shown below.

The key assumptions for this costing model are:

1) The Autonomous Database is always ony started when the data loading workload runs and at the start of each business day. Once the data loading workload has finished and at the end of each business day the Autonomous Database instance is stopped.
2) The number of OCPUs used by each workload are set when the instance is started (either manually or using a script to automate the process).



Loading Data

Workload Description
  • Multiple, concurrent ETL jobs are used to load data from a wide variety of data sources
  • Source data files are located in Oracle Object Store
  • Complex transformations and use of staging tables are part of the ETL process
  • Data loading takes place at night and no other users are on the system
  • Auto-scaling is enabled
Workload Characteristic
  • ETL workload runs for 2 hours 30 minutes every night (5 nights per week)
  • Base workload usage requires 4 OCPUs - 2 hours
  • Peak workload usage requires 12 OCPUs - 30 minutes
  • Per-second billing is used to calculate compute costs

Costing Model for Data Loading Compute Costs

((4 OPCUs * 2 hours * $1.3441) + (12 OPCUs * 30/60 * $1.3441)) * 5 days * 4 weeks

Per month costs for data loading workload

($10.7528 + $8.0646) * 20 days

$376.348

Reporting And Data Analysis

Workload Description
  • Multiple, concurrent queries are run during the day
  • Queries are derived from standard reports, dashboards and adhoc queries
  • Assume no data caching or summary within BI middle-tier server
  • Auto-scaling is enabled
Workload Characteristic
  • Working day is 8 hours per day, 5 days per week
  • The data warehouse is not used at weekends
  • Base query workload usage requires 8 OCPUs - 5 hours
  • Peak morning query workload requires 24 OCPUs - 1:15 hour
  • Afternoon peak query workload requires 16 OCPUs - 45 hours

Costing Model for Query Compute Costs

((8 OPCUs * 5 hours * $1.3441) + (24 OPCUs * 75/60 * $1.3441) + (16 OPCUs * 45/60 * $1.3441)) 5 days * 4 weeks

Per month costs for query loading workload

($53.764 + $40.323 + $16.1292) * 20 days

$2,204.324

Data Storage Costs

Data Storage Requirements
  • Initial raw data set size is 100TB
  • Assume data growth per day of 1TB raw data
  • Assume a compression rate of 8x using EHCC
Factors that do not influence storage costs
  • There is no charge for storing 60 days of online backups
  • There is no charge for putting data in Oracle Object Store

Monthly Costing Model for Initial Data Storage Costs

(100TB * 0.125 * $118.40)

$1,480.00

On-going Monthly Incremental Cost for Data Storage Costs

(1TB * 0.125 * $118.40) * 5 days * 4 weeks

$296.00

Data Discovery Sandbox

Workload Description
  • Data scientist team create clone each month to benchmark existing models
  • Model evaluation and testing of new models takes approx. 2 days (16 hours)
  • Simulataneous testing of multiple models
Workload Characteristic
  • Working day is 8 hours per day
  • The data discovery clone is discarded at the end of the 2-day period
  • Base workload usage requires 4 OCPUs
  • Auto-scaling is not enabled
  • Some additional datasets are accessed directly from Object Store

Costing Model for Query Compute Costs

((4 OPCUs * 16 hours * $1.3441)

$86.0224

Costing Model for Storage Costs

(100TB * 0.125 * $118.40/31/60*16)

$1.0184

ALERT: – Upgrading to Oracle Client Library with Version 19.3 (or later) strongly recommended for your Oracle Autonomous Database

Later this year, in August, there will be changes to the authorization process for client connections to Autonomous Database

DigiCert has recently announced a change in the format of the TLS/SSL certificates that are issued after August 24th, 2022. For your connections to support this new format, we strongly recommend using Oracle client library with version 19.3 or later. You can download the latest versions of these clients from the links provided below:
  • Oracle JDBC Downloads Page
  • Oracle .NET Downloads Page
  • Oracle Instant Client Downloads Page

What does this change mean?

According to the recent announcement made by DigiCert, the public TLS/SSL certificates that are issued after August 24th, 2022 will not contain the Organizational Unit (OU) field.

The TLS/SSL certificates that are used on the server-side of Autonomous Database are issued by DigiCert and are subject to renewal before they expire. The certificates that are renewed after August 24th, 2022 will not have the Organizational Unit (OU) field.

How will this affect my service?

Autonomous Database does server DN matching by default based on the ssl_server_cert_dn property of your connection string, which contains the OU field today. Once the OU field is retired from the server-side certificates, DN matching will not succeed. To avoid any disruptions to existing client connections, Autonomous Database will support doing DN matching based on the host property of your connection string. Since the host name based DN matching is currently supported on clients with version 19.3 or later, we strongly recommend you to upgrade your clients (e.g. JDBC, .NET, Instant Client) to version 19.3 (or later) in preparation for this upcoming change.

Next steps

Please expect to receive further emails from Oracle regarding this change as we approach the deadline set by DigiCert – August 24th, 2022. These emails will help you ensure this necessary change does not impact your ability to connect to your Autonomous Database.

How can I contact Oracle with further questions regarding this announcement?

If you have general questions about preparing for this change, please contact us through the following forums: Autonomous Database for Data Warehousing customer forum or Autonomous Database for Transaction Processing customer forum, respectively. We will reply as soon as possible.

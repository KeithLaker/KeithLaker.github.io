Information About Managing APEX Service

To monitor and manage your APEX Service, find its Database Name in the APEX instances list in Oracle Cloud Infrastructure (OCI) management console. This list is located under Developer Services, APEX Application Development. Here you can:

  • List APEX Service instances
  • Create a new APEX Service instance
  • Launch into APEX or Database Actions on an instance
  • Drill into instance details including APEX usage metrics
  • Access details of the underlying Autonomous Database

Note that your APEX Service instance has special rules because it is Always Free. If there are no HTTPS connections to the instance for 7 consecutive days, it will be automatically stopped. Your data will be preserved, and you can restart the instance to resume using it. If the instance remains stopped for approximately 3 months, it will be reclaimed. We do this to keep Oracle's Always Free cloud environments running clean and provide the best experience for active users. Remember to access your instance periodically to continue using it without interruption for an unlimited time.

If you need to automate lifecycle management of your APEX Service instance, try the Autonomous Database REST APIs. A similar programming interface is available in SDKs and CLI.

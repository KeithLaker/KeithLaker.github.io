Zero-Downtime PL/SQL Changes with Edition-Based Redefinition

Chris Saxon, Developer Advocate, Oracle

10:00 AM - 10:45 AM
Zero-Downtime PL/SQL Changes with Edition-Based Redefinition
10:00 AM - 10:45 AM
Moscone South - Room 307
Releasing code changes to PL/SQL to databases with high uptime requirements brings many challenges. Changing existing packages while the application is running is risky. The workarounds can be difficult and/or expensive to implement. Edition-based redefinition (EBR) enables you to deploy multiple versions of a PL/SQL object in the same schema. Using EBR, you can release PL/SQL easily with zero downtime. This presentation steps through the process of creating and using editions, with a live demo. It also looks at some of the caveats of using EBR and techniques for mitigating these. This session is for DBAs and developers who help with preparing and doing production releases of PL/SQL.

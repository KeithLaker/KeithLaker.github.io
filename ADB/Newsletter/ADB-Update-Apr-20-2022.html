SIMPLER MANAGEMENT


DATABASE LINKS

NEW - Create DB links over TCP to targets on private endpoints


SIMPLER MANAGEMENT

CHALLENGE
Where the target database for a DB link is an on-premise Oracle Database or a cloud service instance (e.g. DBCS, bare metal, etc. etc.), it may not be configured for TCPS authentication. Configuring TCPS and setting up the wallet takes time and effort and is not straightforward for most DBAs.

SOLUTION
ADB-Shared now supports database links to the target databases on a private endpoint which are not already configured to use TCPS authentication. For fleet and cloud admins, this simplifies creating and managing database links to other databases within a PE configuration.





This feature is not enabled by default! If customers want to use this feature, they need to file an SR at Oracle Support and request it to be enabled for their database.

SERVICE LEVELS

NEW - Availability Service Level Objectives For ADB-Shared


SIMPLER MANAGEMENT

CHALLENGE
Customers running important workloads need to understand the level of protection provided by ADB-Shared for their applications and data during unplanned outages or planned maintenance activities.

SOLUTION
ADB-Shared is engineered to return an application online following an unplanned outage or a planned maintenance activity within seconds. Publicly available Target Recovery Time Objective (RTO) SLOs for different types of failure events.




LEARN MORE

DOC: Availability Service Level Objectives (SLOs), click here


BUSINESS VALUE


Snowflake has no published RTO/MTTR

AWS offers range of RPO/RTO scenarios from +24 hours to near-zero, depending on different deployment scenarios and customer’s budget

SQL Server has documented RPO/RTO but requires Business Critical tier and geo-replication to provide a guarantee of RPO of 5 sec and RTO of 30 sec for 100% of deployed hours.

FASTER INNOVATION


DATA LAKEHOUSE

NEW: Smarter options for synchronization with Data Catalog


FASTER INNOVATION

CHALLENGE
Data engineers, data scientists etc., must be able to quickly discover new data sets and consume updates to the structure and organization of existing data sets.

As the breadth of services within the data lakehouse grows, teams need a flexible process for synchronizing Autonomous Database with OCI Data Catalog.

SOLUTION
DBMS_DCAT RUN_SYNC procedure has new parameters that provide more control over how the synchronization is performed:

- refresh to capture the latest changes
- replace to run full refresh

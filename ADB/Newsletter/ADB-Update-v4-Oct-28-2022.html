NEW FEATURE ANNOUNCEMENTS


DATA SHARING

NEW Making it easier to share and consume data sets across companies and across regions / data centers


DATA SHARING

KEY SALES PLAYS

DATA WAREHOUSING
AND DATA LAKE

MODERNIZE WITH
DATABASE 19c

CLOUD NATIVE APPS

OVERVIEW
There are two parts to our data-sharing announcement.

1) Making it much easier to share data between Autonomous Databases:

  • Share across companies and across regions / data centers
  • Secure and governed data sharing: grant, revoke, audit, track
  • No replication required

2) As part of a much broader data sharing strategy, Oracle is adopting open-standards-based data sharing with Delta Share API. This approach allows Autonomous Database to act as both:

  • Data Provider
    • Share data with any compliant Data Share Data Recipient
    • Versioned data share support for Oracle Object Store
    • Latest data share support for database objects (Oracle-to-Oracle)
    • Share management through API and UI
  • Data Recipient
    • Consume any Delta Share from any compliant provider
    • Data share access through external tables


STATUS: Previews for this feature will be coming shortly. If you have customers that are actively looking for data-sharing capabilities then please reach out to the product management team to discuss this in more detail.

BUSINESS VALUE


Having a strong data-sharing message in the data warehouse/data lake space for our customers is extremely important.

Our approach in this area is very different from Snowflake whose objective is to lock their customers and data partners into the Snowflake platform.

By adopting an open-standards approach we are allowing our customers to use any tool or service on any cloud that supports the delta-sharing standard to both consume and share data with Autonomous Database. THERE IS NO LOCK-IN!

ENHANCEMENTS TO EXISTING FEATURES


AUTONOMOUS DATABASE FOR DATA LAKES

Simple, more secure data file access across all clouds for data lake use cases


DATA LAKE

KEY SALES PLAYS

DATA WAREHOUSING
AND DATA LAKE

MODERNIZE WITH
DATABASE 19c

OVERVIEW
Autonomous Database makes it possible to quickly and easily create data lakes where SQL queries run directly on data stored in object storage. A wide range of data formats are supported such as CSV, Parquet, ORC, JSON, Avro etc.

Autonomous Database connects to all cloud data file stores making it possible to query or load data from OCI, AWS, Azure, and GCP.

Latest enhancements mean there is no need for cloud user credentials since it is now possible to use vendor-specific cloud access policies to provide granular access control. There is a simple API to enable access:

DBMS_CLOUD_ADMIN.ENABLE_PRINCIPAL_AUTH(provider => ‘OCI/Azure/AWS/GCP’, …)

DATABASE ACTIONS - DATA STUDIO

Enhancements for Autonomous Data Warehouse’s built-in tool suite


DATA STUDIO

KEY SALES PLAYS

DATA WAREHOUSING
AND DATA LAKE

MODERNIZE WITH
DATABASE 19c

CLOUD NATIVE APPS

OVERVIEW
The built-in Database Actions UI has been enhanced and now includes a Data Studio.

There is a new integrated workflow interface for business analysts which spans data loading, data transformations, and analysis. These are now all grouped together within Data Studio.

Flexible Data Load

  • Load data from files on your computer or from cloud storage
  • Copy the data, link to the data to query it in place, or enable a continuous feed of the data into tables

Data Analysis

  • Design Analytic Views to provide a common way of analyzing data over multiple linked tables. Examine the data graphically in the tool to check it makes sense
  • Data issues that inhibit data analysis are flagged automatically

Automatic Insights

  • Our algorithms automate the search for the needle in your haystack of data
  • Identify a column of interest and we will search for anomalies in it

Catalog

  • Explore the contents of your database, or search for objects of interest
  • Understand the dependencies among those objects -- where the data came from and what else uses it

DATA PIPELINES

NEW Data pipelines for loading and exporting data


SIMPLER MANAGEMENT


KEY SALES PLAYS

DATA WAREHOUSING
AND DATA LAKE

MODERNIZE WITH
DATABASE 19c

CLOUD-NATIVE APPS

OVERVIEW
Data pipelines are a new built-in capability that provides a simple way to continuously and incrementally export and import data into ADB.

Data load pipelines: Used for periodically loading data into the database from new data files in object store. Typical use cases include:

  • Continuous migration of new on-premise data sets into a database from the object store of choice
  • Real-time loading of new data via object storage from ETL/streaming process to support real-time data analytics

Data export pipelines:
Used to periodically export new, incremental data as results from a table or query to the object store. Typical use cases include:

  • Incremental export at specific intervals to object store of time-series style data generated by an application

Oracle-managed pipelines
built-in pipelines include a set of pre-defined pipelines for managing audit logs and APEX activity logs.

BUSINESS BENEFIT
Provides a built-in, flexible framework for managing the creation of real-time data, pushing data to object storage at pre-defined points in time, along with a simpler hands-free way to load data into an Autonomous Database.

This removes the need to install and/or use additional tools or cloud services to manage data integration in real time or through incremental updates.

This new feature provides ADB with similar data pipeline features to those offered by competitors such as Snowflake and Redshift for data warehouse and data lake projects.

CHANGE TO SERVICE LIMITS FOR MEDIUM AND HIGH

UPDATE - Enabling auto-scaling increases service limits for medium and high


SIMPLER MANAGEMENT

KEY SALES PLAYS

DATA WAREHOUSING
AND DATA LAKE

MODERNIZE WITH
DATABASE 19c

CLOUD-NATIVE APPS

OVERVIEW
Autonomous Database makes life much simpler for the cloud admin. ADB has a predefined set of service names to support different workloads and workload priorities.



If auto-scaling is enabled, then HIGH and MEDIUM services offer much higher concurrency. Additional control is provided for cases where selecting one of the default services does not meet an application’s performance needs. The parameters for the MEDIUM service can be modified.




BUSINESS BENEFIT
Enabling auto-scaling increases service limits for medium and high making it possible to get more done!

Most competitor databases have minimal workload management capabilities, and concurrency is typically low. Many DBAs are forced to invest a lot of time and effort to manage concurrency and workloads.

Snowflake has fixed concurrency of 8 per cluster (virtual data warehouse). Snowflake DBA has to monitor and manage the cluster scaling process to keep costs down.

By default, Amazon’s database services have very poor concurrency, but DBAs can use various tools to increase concurrency levels depending on the type of workload. User experience claims there is a maximum ceiling of around 50 concurrent queries.

INTEGRATION WITH SLACK

NEW - Send notifications and query results to a Slack channel


SIMPLER MANAGEMENT

KEY SALES PLAYS

DATA WAREHOUSING
AND DATA LAKE

MODERNIZE WITH
DATABASE 19c

CLOUD-NATIVE APPS

OVERVIEW
Autonomous Database supports sending messages/notifications from Autonomous Database to a specific Slack channel. Use the DBMS_CLOUD_NOTIFICATION package to send messages to a Slack channel:

BEGIN
DBMS_CLOUD_NOTIFICATION.SEND_MESSAGE(
provider => 'slack',
credential_name => 'SLACK_CRED',
message => 'Alert from Autonomous Database...',
params => json_object('channel' value 'C0....08'));
END;

Use the same package to send output from a query to a Slack channel:

BEGIN
DBMS_CLOUD_NOTIFICATION.SEND_DATA(
provider => 'slack',
credential_name => 'SLACK_CRED',
query => 'SELECT username, account_status, expiry_date FROM account_users',
params => json_object('channel' value 'C0....08',
'type' value 'csv'));
END;

BUSINESS BENEFIT
Slack is an integral part of many organizations and is a key channel for sharing information. Many tools can already interact with and push messages to Slack, e.g. Microsoft Outlook. Cloud users can send messages, alerts, or output of a query from an Autonomous Database to a Slack channel.

AUTOMATIC UPDATES FOR TIMEZONE FILES

Option to automatically upgrade an instance to use the latest timezone files


EASIER MIGRATIONS

KEY SALES PLAYS

DATA WAREHOUSING
AND DATA LAKE

MODERNIZE WITH
DATABASE 19c

CLOUD-NATIVE APPS


OVERVIEW
Common migration issue - mismatched time zone files when importing Data Pump file:

ORA-39405: Oracle Data Pump does not support importing from a source database with TSTZ version n+1 into a target database with TSTZ version n.

Time zone files for ADB are periodically updated to reflect the latest time zone-specific changes. ADB can be optionally configured to use the latest time zone files automatically – note: this is not enabled by default.

Enabling this feature means starting/restarting an ADB instance takes slightly longer due to the additional processing.

BUSINESS BENEFIT
Time zone file automatic upgrade process simplifies the migration process to ADB when using data pump.

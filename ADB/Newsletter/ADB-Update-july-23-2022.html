SIMPLER MIGRATIONS


ADB + RAT

NEW - ADB now supports Real Application Testing.


SIMPLER MIGRATIONS

CHALLENGE
Many customers try to test the effects of software and infrastructure changes on their existing workloads using simulated data sets. The challenge is that these simulated data sets do not accurately represent their actual production workloads. Therefore, many customers are reluctant to adopt new technologies, such as Autonomous Database, and adapt their businesses to the rapidly changing competitive pressures.

SOLUTION
Real Application Testing helps customers thoroughly assess the effect of software and infrastructure changes on real-world application workloads within a test environment before deploying those changes to a production system.

Oracle Real Application Testing consists of two features, Database Replay and SQL Performance Analyzer. Together they allow customers to rapidly adopt new technologies, such as Autonomous Database, that add significant value to the business while minimizing risk.




LEARN MORE

DOC: Oracle Real Application Testing, click here.




BUSINESS VALUE


Customers can now capture a workload from an on-premises or other cloud service database and replay it on an Autonomous Database instance. This makes it very easy to compare workloads between an on-premises or other cloud service database and an Autonomous Database.



 


ORACLE JVM

NEW - ADB now supports embedded Java Virtual Machine (JVM)


SIMPLER MIGRATIONS

CHALLENGE
Many customers leverage the embedded Oracle JVM within the Oracle Database. Those customers want the benefits of an Autonomous Database without changing their existing application architecture. Until now, Autonomous Database did not support Oracle Java, preventing many customers from migrating to ADB.

SOLUTION
Autonomous Database supports Oracle JVM. Oracle JVM is a standard, Java-compatible environment that runs any pure Java application.

Oracle JVM is compatible with the standard JLS and the JVM specifications. It supports the standard Java binary format and the standard Java APIs. In addition, Oracle Database adheres to standard Java language semantics, including dynamic class loading at run time.




LEARN MORE

DOC: Use Oracle Java, click here.




BUSINESS VALUE


This makes it even easier for Oracle Database customers to move their existing workloads to Autonomous Database so they can gain the following benefits:

  • Lower costs with autoscaling and per-second billing
  • Reduce risk with self-securing features and Oracle Data Safe
  • Innovate more with Oracle Machine Learning
  • Support high availability requirements with SLAs covering complete database service lifecycles

STRONGER SECURITY


WALLET ROTATION

NEW - Option to rotate wallet with a grace period


STRONGER SECURITY

CHALLENGE
Customers use the wallet rotation feature to invalidate existing client certification keys for their Autonomous Database instances within a region. However, it can be challenging to immediately update all applications and tools to use the new wallet. Ideally, customers want a set window when existing apps and tools can continue to use the existing wallet, avoiding interruptions to existing services and workloads.

SOLUTION
Autonomous Database now allows customers to choose between immediate rotation or a grace period of 1 to 24 hours when they rotate their wallets.

During the grace period, customers can inform other teams to download the new wallet and update their applications and tools to use the new wallet. The old and new client certification keys are valid during the grace period. Autonomous Database invalidates the old client certification keys when the grace period expires, and only the new client certification keys are valid.




LEARN MORE

DOC: Rotate wallets with a grace period, click here


BUSINESS VALUE


Setting a grace period allows customers to perform wallet rotation without incurring downtime.

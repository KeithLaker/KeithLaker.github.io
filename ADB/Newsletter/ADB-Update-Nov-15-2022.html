SIMPLER MANAGEMENT



CROSS-REGION REFRESHABLE CLONES

NEW Create a clone in a different region to the primary (source) database.


SIMPLER MANAGEMENT


KEY SALES PLAYS

DATA WAREHOUSING
AND DATA LAKE

MODERNIZE WITH
DATABASE 19c

CLOUD-NATIVE APPS

OVERVIEW
Autonomous Database provides cloning where you can choose to create a full clone of the active instance, create a metadata clone, or create a refreshable clone. With a refreshable clone the system creates a clone that can be easily updated with changes from the source database. Typical use cases include:

  • Create copies of primary database for use as read-only reporting
  • Create one or more clones of a primary database in other regions
  • Share copies of a production database with multiple business units
  • Assign cost of database usage across multiple business units
  • Use a refreshable clone as a dev/test/training database

BUSINESS BENEFIT
Position refreshable clones as a simple way for development teams to create a test database where they can disconnect a refreshable clone from its source and run DML operations, calculations, and queries as needed. This allows them to make changes while the database is disconnected and when testing is completed, simply reconnect to the source database, which refreshes the clone to the point when it was disconnected.

Position refreshable clones as a simple way for development teams to share data across multiple teams and charge each individual team for their usage.

OPTIMIZATIONS FOR BULK OPERATIONS ON FILES

NEW - DBMS_CLOUD offers parallel execution support for bulk file upload, download, copy, and transfer activities


SIMPLER MANAGEMENT


KEY SALES PLAYS

DATA WAREHOUSING
AND DATA LAKE

MODERNIZE WITH
DATABASE 19c

CLOUD-NATIVE APPS

OVERVIEW
Enhancements to PL/SQL package DBMS_CLOUD mean that it now offers parallel execution support for bulk file operations. This streamlines the user experience and delivers optimal performance for bulk file operations.

DBMS_CLOUD bulk file operations support downloading and uploading files on Autonomous Database, including support for copying, moving, and deleting files across cloud object stores (OCI Object Storage, Amazon S3, Amazon S3-Compatible, Azure Blob Storage, Google Cloud Store), either within the same object store provider or between Object Store providers.

BUSINESS VALUE
This feature internalizes and automates the optimizations for bulk file operations meaning all users can benefit from faster performance and more efficient use of available resources.

DEFER APEX UPGRADE

APEX customers now have a lot more control over the upgrade process


AGILE DEVELOPMENT

KEY SALES PLAYS

DATA WAREHOUSING
AND DATA LAKE

MODERNIZE WITH
DATABASE 19c

CLOUD-NATIVE APPS


OVERVIEW
In the past, Oracle automatically deployed APEX releases to ADB and APEX Service databases as soon as possible. Customers could opt for a temporary upgrade deferral period of up to 45 days to prepare for the upgrade and apply it at a time of their choosing.

Starting with the APEX 22.2 release, the 45-day time window becomes the default for all databases - with no opt-in required.

Customers with paid databases can choose to extend their upgrade window from 45 to 90 days. This setting can be changed in APEX Administration Services and changes will take effect for the next upgrade cycle. Existing paid databases that are configured to defer upgrades will be automatically set to the new longer window of 90 days.

BUSINESS VALUE
APEX customers now have significant control over the upgrade process with up to 90 days to evaluate and test any planned upgrades. Cloud and fleet admins can use the new PL/SQL API to check if an APEX upgrade is available.

SEARCHING TEXT DATA

Using a text index to optimize querying text data on object storage


DATA LAKE

KEY SALES PLAYS

DATA WAREHOUSING
AND DATA LAKE

MODERNIZE WITH
DATABASE 19c


OVERVIEW
DBMS_CLOUD supports building a text index on the object store files. This makes it possible to use wildcards in text searches. Support for word-based search works for commonly used data formats, for example CSV and JSON. It is possible to configure a refresh rate that indicates the frequency in minutes at which the index is refreshed for any new uploads or deletes.

BUSINESS BENEFIT
This makes it quicker and easier to use sophisticated text searches, such as use wildcard searches, on data sets within a data lake.

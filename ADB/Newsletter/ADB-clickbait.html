AUTONOMOUS DATABASE
FACT 2
Autonomous Database Keeps Storage Costs Low and Transparent

A Snowflake table is built around a series of micro-partitions where each one is 16MB in size. Snowflake's architecture means that updates force new micro-partitions to be created for each update. The old, out-of-date data micro-partitions are not removed but continue to consume storage space. Snowflake customers can experience rapid storage bloat when doing even small scale updates to their data warehouse.



Due to its robust architecture, Autonomous Database does not suffer from storage bloat during an update process. In real world scenarios, Autonomous Database customers need upto 31x less storage space.

PROVE IT!

Oracle's robust data architecture means that Autonomous Database customers need upto 31x less storage space for real-world data warehouses.

AUTONOMOUS DATABASE
FACT 3
Autonomous Database is processes real-world workloads 23x faster

Since Snowflake can only create new micro-partitions this means there are no internal mechanisms to efficiently track and update existing data points. Snowflake's believes that if existing data needs to be updated, it is always quicker to delete the existing data and completely reload the updated data. This approach has significant drawbacks in terms of performance and ease of use.



Autonomous Database has Oracle’s industry-leading database technology at its core, which contains a very sophisticated data management engine. Establishing business rules to enforce data integrity creates database structures that also help to speed up updates. Autonomous Database can process real-world update operations up to 23x faster.

Autonomous Database ensures data is always 100% trustworthy

Snowflake only supports the most basic data integrity rule - preventing null, or empty, rows from being loaded. It is possible for dimension tables and fact tables to contain duplicate rows making it impossible to fully trust data shown in reports or charts.




Oracle Autonomous Database provides the most comprehensive range of data constraints and business rules covering unique rows, and relationships between columns in different tables to ensure data is completely trustworthy.

RECOVERING FAILED TRANSACTIONS

NEW - OraMTS recovery feature on Autonomous Database


SIMPLER MANAGEMENT

CHALLENGE
Many Microsoft customers use Microsoft Transaction Server combined with Oracle Database. In moving their data management platform to Autonomous Database developers and IT, teams need a robust way to manage in-doubt or failed transactions resulting from a failure in one or any of the following layers: Microsoft Transaction Server application, network, and/or Microsoft Distributed Transaction Coordinator.

SOLUTION
Oracle MTS (OraMTS) Recovery Service resolves in-doubt transactions on the computer that started the failed transaction. A scheduled recovery job for each MTS-enabled database lets the OraMTS Recovery Service resolve in-doubt transactions.




LEARN MORE

DOC: Using OraMTS Recovery Feature on Autonomous Database, click here.




BUSINESS VALUE


Support OraMTS Recovery Feature on Autonomous Database enables the delivery of robust and highly available applications built around a multi-cloud architecture.

SMARTER ANALYTICS


OML4R

NEW - Work faster and smarter by using R with ADB


SMARTER ANALYTICS

CHALLENGE
Data scientists and other R users need the ability to explore, prepare, and analyze data at scale using the native R APIs.

They are looking for ways to eliminate or reduce data movement and latency to enable a faster, more agile approach to model building and deployment and get faster time-to-value.

SOLUTION
OML4R allows R users to leverage ADB as an HPC engine to explore and prepare data, build models, score data, and deploy solutions - all using R.

Store and manage R objects and user-defined R functions in the database - avoiding the need to manage separate files.








LEARN MORE

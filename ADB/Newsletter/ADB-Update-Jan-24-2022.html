REDUCING RISK
IAM AUTHENTICATION
(Shared Infrastructure Only)
Integration With Identity and Access Management (IAM) Authentication

We are excited to announce the integration of Oracle Autonomous Database - Shared (ADB-Shared) with OCI IAM. This integration will improve the ADB-Shared user experience in three ways.

First, administrators can now manage all their ADB-Shared users centrally, without making changes to each database individually when users join, change roles, or leave an organization. Second, database users can log in using their single IAM database credential instead of remembering and using a different password for each database.

Finally, tools or applications that accept OCI IAM tokens can now use single sign-on (SSO) to connect the end-user to the database.

SIMPLER MANAGEMENT
WALLETLESS CONNECTIONS
(Shared Infrastructure Only)
Walletless connection on Linux x64 Clients using Oracle Call Interface

Using TLS authentication for Oracle Call Interface connections on Linux x64, a wallet is not required when the client program connects with Oracle Instant Client 19.13.

Using TLS authentication for Oracle Call Interface connections on all other platforms, or without Oracle Instant Client 19.13, clients must provide a generic CA root certificate wallet.

DATABASE LINKS
(Shared Infrastructure Only)




Use Database Links with a Target Database on a Private Endpoint

You can now create database links from an Autonomous Database source to a target Oracle Database on a private endpoint.

To create a database link to a target database on a private endpoint, the target database must be accessible from the source database's Oracle Cloud Infrastructure VCN.

SIMPLER MANAGEMENT
REFRESHABLE CLONES
(Shared Infrastructure Only)




Reconnect a Disconnected Refreshable Clone

After you disconnect a refreshable clone, during the following 24 hour period, you can now reconnect that refreshable clone to the source database. This feature allows you to use a refreshable clone as a test database, run DML, and make changes while the database is disconnected.

When you are done with your testing, you can reconnect to the source database, which refreshes the clone to the point where you disconnected.





LEARN

DOC: Reconnect a Refreshable Clone to the Source Database, click here.

BLOG: Keep your clone's data up-to-date with Refreshable Clones in Autonomous Database, click here.



VALUE

This feature makes it easier to create and manage clones for testing, QA, training, etc.

CROSS-REGION CLONES
(Shared Infrastructure Only)




ADB Now Supports Cross-Region Clones

When you clone an Autonomous Database instance using the Oracle Cloud Infrastructure Console, you can select your preferred region for the clone. You can create the clone in the current region or a remote region. The list of available regions to create a clone only shows the regions to which you are subscribed.

REDUCING COSTS
ADB STOP - START SCHEDULE
(Shared Infrastructure Only)




Defining A Calendar Schedule to Stop and Start An ADB

It is now possible to schedule start and stop times for an Autonomous Database instance. When you enable Auto Start/Stop Schedule on an Autonomous Database instance, the instance automatically starts and stops according to the specified schedule. This feature allows you to reduce costs by scheduling shutdown periods when a system is not in use.




LEARN

VALUE

This allows customers to reduce costs by scheduling shutdown periods for times when an ADB is not in use.

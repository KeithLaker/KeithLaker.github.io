ADB-Shared Now Available In All US Governmental Clouds

ADB-Shared now available in all US Federal cloud regions with DISA Impact Level 5 authorization.

US Federal Cloud with DISA Impact Level 5 authorization supports applications that require Impact Level 5 (IL5) data, as defined in the Department of Defense Cloud Computing Security Requirements Guide.

REDUCING RISK
IAM AUTHENTICATION
(Shared Infrastructure Only)
Integration With Identity and Access Management (IAM) Authentication

OCI Identity and Access Management users can now authenticate and authorize to ADB-Shared.

Centrally managing users is key to better security since user access to databases is managed centrally instead of locally in every database. It also reduces zombie database user accounts that are forgotten and left behind when a user leaves an organization. User management moves from the long list of DBA tasks to the IAM administrator.

Users with SQL*Plus and similar tools can sign into Autonomous Database using their IAM username and IAM database password and continue using existing supported database clients.

Users can also use IAM SSO tokens with the latest JDBC-thin and OCI-C database clients to connect with ADB-Shared.

SIMPLER MANAGEMENT
WALLET-LESS CONNECTIONS
(Shared Infrastructure Only)
Wallet-less connections on Linux x64 clients using Oracle Call Interface

Using TLS authentication for Oracle Call Interface connections on Linux x64, a wallet is not required when the client program connects with Oracle Instant Client 19.13.

Using TLS authentication for Oracle Call Interface connections on all other platforms, or without Oracle Instant Client 19.13, clients must provide a generic CA root certificate wallet.

VALUE

Simplifies the process for connecting additional tools and services to Autonomous Database.

SIMPLER MANAGEMENT
GET MY IP ADDRESS
(Shared Infrastructure Only)
New UI Button To Quickly Add Your IP Address To ACLs

This UI enhancement makes it easier to setup ACLs where you need to add the IP address of the client to your Network Access Control List (ACL). The button "Add My IP Address" adds your current IP address to the ACL entry.

This removes the need to manually look for your client IP via a 3rd party website or app anymore (e.g. Google, whatsmyip.com etc).

This new button is available in both the Create Autonomous Database flow for new ADB-S instances to be provisioned and the Update Network Access flow for existing ADB-S instances.

SIMPLER MANAGEMENT
REFRESHABLE CLONES
(Shared Infrastructure Only)




Reconnect a Disconnected Refreshable Clone

After you disconnect a refreshable clone, during the following 24-hour period you can now reconnect that refreshable clone to the source database. This feature allows you to use a refreshable clone as a test database, run DML, and make changes while the database is disconnected.

When you are done with your testing, you can reconnect to the source database, which refreshes the clone to the point where you disconnected.





LEARN

DOC: Reconnect a Refreshable Clone to the Source Database, click here.

BLOG: Keep your clone's data up-to-date with Refreshable Clones in Autonomous Database, click here.



VALUE

This feature makes it easier to create and manage clones for testing, QA, training, etc.

ADB STOP - START SCHEDULE
(Shared Infrastructure Only)




Defining A Calendar Schedule to Stop and Start An ADB

It is now possible to schedule start and stop times for an Autonomous Database instance. When you enable Auto Start/Stop Schedule on an Autonomous Database instance, the instance automatically starts and stops according to the specified schedule. This feature allows you to reduce costs by scheduling shutdown periods when a system is not in use.

SIMPLER MANAGEMENT
MAINTENANCE INFORMATION
(Shared Infrastructure Only)


View Maintenance Status and Timezone Version Notifications

The DB_NOTIFICATIONS view stores information about maintenance status notifications and timezone version upgrade notifications for your Autonomous Database instance.

Developers and DBAs can incorporate this information into their application/program code/scripts to programtically stop/pause long running jobs before maintenance events start and then restart the job after maintenance event is complete.



LEARN

DOC: View Maintenance Status and Timezone Version Notifications, click here.



VALUE

This gives fleet and cloud admins a quick and easy way to get detailed information about the patching information and the status of the timezone file used by their ADB instance.

This information could be used to automatically and gracefully stop/pause long running jobs before maintenance events start and then restart the job after maintenance is complete - in real-time.

DATABASE AVAILABILITY

NEW - View monthly database availability


SIMPLER MANAGEMENT

CHALLENGE
Customers need to have the ability to view the availability information for their Autonomous Database instances.

SOLUTION
Availability information is now available directly on the OCI console




LEARN MORE

DOC: Availability Service Level Objectives (SLOs), click here


BUSINESS VALUE


Snowflake has no published RTO/MTTR

AWS offers range of RPO/RTO scenarios from +24 hours to near-zero, depending on different deployment scenarios and customer’s budget

SQL Server has documented RPO/RTO but requires Business Critical tier and geo-replication to provide a guarantee of RPO of 5 sec and RTO of 30 sec for 100% of deployed hours.



 


WORKING WITH AZURE

NEW - Simplified access to resources on Azure


SIMPLER MANAGEMENT

CHALLENGE
Customers need a simple way to establish a bridge between Autonomous Database and resources that live in Microsoft Azure. The requirement is to remove the need to create or store individual credentials in Autonomous Database.

SOLUTION
Autonomous Database now supports Azure service principal authentication. Azure service principal authentication allows Autonomous Database to access Azure services without needing to provide long-term credentials.

Eliminates need for users to create/store individual credentials or access key/tokens in the database.




LEARN MORE

DOC Use Azure Service Principal to Access Azure Resources, click here

BLOG How to Easily Access Azure Resources from Your Autonomous Database, click here



 


DATABASE GATEWAYS

UPDATE - Connect to Azure SQL and Azure Synapse Analytics using built-in heterogeneous connectivity


SIMPLER MANAGEMENT

CHALLENGE
Many customers have data stored in non-Oracle databases and want to access these data sets quickly and efficiently. Oracle provides Database Gateway to transparently access data residing in a non-Oracle system from an Oracle environment. However, setting up and configuring this type of capability can take a lot of time and effort.

SOLUTION
Autonomous Database now includes heterogeneous connectivity for Azure SQL and Azure Synapse Analytics.




LEARN MORE

DOC: Create Database Links to Non-Oracle Databases with Oracle-Managed Heterogeneous Connectivity, click here

BLOG: Access Amazon Redshift, Snowflake, and Other Non-Oracle Databases from Your Autonomous Database, click here




BUSINESS VALUE


This feature makes it very easy for technical and non-technical users to access data in non-Oracle databases such as Azure SQL, Azure Synapse Analytics, Snowflake, Redshift, MySQL, and PostgreSQL. Customers can then query their data directly from Autonomous Database without moving it, allowing them to innovate faster. Alternatively, customers can use this connectivity as a simple migration service to help them move their data sets to Autonomous Database.

Use this new feature to target Snowflake customers experiencing extremely high running costs and poor performance and looking for a lower cost, more productive, faster data warehouse platform.

DATABASE NAMES

NEW - Create longer database names!


SIMPLER MANAGEMENT

CHALLENGE
Customers have been asking for more flexibility for database names as the current 14-character limit is too restrictive for many use cases and/or internal project naming conventions.

SOLUTION
The length limit for database names when you create or clone your database has been increased from 14 to 30 characters.

Eliminates need for users to create/store individual credentials or access key/tokens in the database.

ADB-Shared feature roadmap for next 6-9 months


COMING SOON

Here is a sneak peek into features coming to Autonomous Database within the next 6-9 calendar months. The list of features is shown in alphabetic order - not release data order!

    • dbms_cloud to support Google principal authentication

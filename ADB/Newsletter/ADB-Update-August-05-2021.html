AUTOMATIC PARTITIONING
(Shared Infrastructure Only)
Automatic Partitioning For Tables And Indexes

Automatic partitioning analyzes and automates partition creation for tables and indexes of a specified schema to improve performance and manageability in Autonomous Database. Automatic partitioning, when applied, is transparent and does not require any user interaction or maintenance.

PATCH DETAILS
(Shared Infrastructure Only)
Database View To List Patch Details

You can view Autonomous Database maintenance event history to see details about past maintenance events via the Autonomous Database Details page under 'Maintenance' then click 'View History'. The information listed includes a start time and stop time for past maintenance events.



To view the detailed patch information for a specific patch use the Title field, for example:

SELECT * FROM DBA_CLOUD_PATCH_INFO WHERE PATCH_VERSION = 'ADBS-21.7.1.1';



LEARN

DOC: View Patch Details, click here



POSITION

Position with fleet and cloud admins who want more information about the patches applied to their ADB instances.

EARLY ACCESS TO PATCHES
(Shared Infrastructure Only)
Autonomous Database Now Provides Early Access To Patches

When you provision or clone an Autonomous Database instance you can select a patch level to apply upcoming patches. There are two patch level options: Regular and Early. The Early patch level allows customers to test upcoming patches one week before they are applied as part of the regular patching program.





The console shows the patch level setting with the section headed Maintenance.





LEARN

DOC: Set the Patch Level, click here



POSITION

Position with fleet and cloud admins who want early access to upcoming patches for testing purposes.

LiveLabs Workshop To Expose Snowflake Weaknesses

We have released two new workshops to help you and our customers learn more about the weaknesses in Snowflake's architecture and analytical capabilities. This first workshop focuses on a real-world data loading scenario that demonstrates the unique capabilities of Oracle's data management capabilities. In comparison, customers will see how Snowflake lacks data integrity, and its storage requirements can unexpectedly and rapidly expand, so they are hit with unexpected costs.

This second workshop demonstrates some of the unique analytical capabilities of Autonomous Data Warehouse that are missing in Snowflake.

CAPACITY PLANNING

NEW - Autonomous Database now supported in Oracle Cloud Capacity Analytics sizing tool


SIMPLER MANAGEMENT

The following is a summary of the email sent by Chris Craft (Senior Director, Product Management Systems Technology), with the subject line "OCCA Sizing Tool - Support for Autonomous Database - Shared", dated March 3, 2022...

Oracle Cloud Capacity Analytics (OCCA) now includes sizing for Autonomous Database - Shared. OCCA sizes database platforms based on historical consumption metrics collected for each database via an extract from Oracle Enterprise Manager or Oracle Database Automatic Workload Repository (AWR).

DATABASE GATEWAYS

NEW - Connect to Non-Oracle databases using built-in heterogeneous connectivity


SIMPLER MANAGEMENT

CHALLENGE
Many customers have data stored in non-Oracle databases and want to access these data sets quickly and efficiently. Oracle provides Database Gateway to transparently access data residing in a non-Oracle system from an Oracle environment. However, setting up and configuring this type of capability can take a lot of time and effort.

SOLUTION
Autonomous Database support for Oracle-managed heterogeneous connectivity makes it easy to create database links to non-Oracle databases. When you use database links with Oracle-managed heterogeneous connectivity, Autonomous Database configures and sets up the connection to the non-Oracle database. The following connection types are supported with Autonomous Database:

  • Amazon Redshift
  • MySQL
  • PostgreSQL
  • Snowflake

BUSINESS VALUE


This feature makes it very easy for technical and non-technical users to access data in non-Oracle databases such as Snowflake, Redshift, MySQL, and PostgreSQL. Customers can then query their data directly from Autonomous Database without moving it, allowing them to innovate faster. Alternatively, customers can use this connectivity as a simple migration service to help them move their data sets to Autonomous Database.

Use this new feature to target Snowflake customers experiencing extremely high running costs and poor performance and looking for a lower cost, more productive, faster data warehouse platform.



 


EXTERNAL TABLES

NEW - External tables with partitioning specified in source files


SIMPLER MANAGEMENT

CHALLENGE
Using partitioned external tables can dramatically improve query performance by only accessing the data required for the query. However, creating the DDL for each partition and maintaining the list of valid partitions can be very time-consuming and complicated.

SOLUTION
This latest update to ADB introduces additional capabilities to DBMS_CLOUD.CREATE_EXTERNAL_PART_TABLE. Using the file partitioning specified in source files, Autonomous Database can analyze the Cloud Object Store file path information to determine the correct partition columns and data types and create the required definitions for each partition.

This feature is incorporated into the Database Actions Data Loading tools so business users can benefit from this feature within their self-service data tools.

This is an important "simpler management" feature for all data lakehouse opportunities.

IAM AUTHENTICATION

FASTER INNOVATION


CODE REPOSITORIES

NEW: Use a Cloud Code Repository with Autonomous Database


FASTER INNOVATION

CHALLENGE
Developers use code repositories to manage their application code. They need an easy way to pull and push their code in their cloud code repository directly into and out of Autonomous Database.

SOLUTION
Autonomous Database now provides routines that allow developers and general users to manage and store their code files in Cloud Code (Git) Repositories. Supported cloud code repositories are:

  • GitHub Repository
  • AWS CodeCommit
  • Azure Repos

The DBMS_CLOUD_REPO package provides a single interface for accessing a Cloud Code Repository from Autonomous Database. It provides the following to help developers work with their cloud code repositories:

  • Repository initialization operations allow you to initialize a repository.
  • Repository management operations let you create, list, update or delete a repository.
  • Repository file management operations to upload, download, update, and delete files.
  • SQL install operations that let you export database object metadata DDL to a repository and install SQL statements into the database from a Cloud Code Repository.



LEARN MORE

DOC: Using and Managing a Cloud Code Repository with Autonomous Database, click here.




BUSINESS VALUE


Makes it easier for developers and cloud DBAs to leverage their cloud repository to manage their application code and scripts from within Autonomous Database. This will enable faster and smarter ways to work with application code and scripts, leading to faster innovation.

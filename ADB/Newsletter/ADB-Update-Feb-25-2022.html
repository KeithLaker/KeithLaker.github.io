SIMPLER MANAGEMENT
(Shared Infrastructure Only)
Built-in views to query OCI resource, cost, and usage data

CHALLENGE: Customers can download their cost and usage data directly from the OCI console in csv file format. However, getting to the stage where you can slice and dice and analyze that data can involve a lot of steps.

SOLUTION: Autonomous Database now has built-in views that show OCI resources, cost and usage data. You can access this OCI information using the following views:

  • OCI_AUTONOMOUS_DATABASES
  • OCI_BUDGET_ALERT_RULES
  • OCI_BUDGET_SUMMARY
  • OCI_COST_DATA View
  • OCI_OBJECTSTORAGE_BUCKETS
  • OCI_USAGE_DATA
BUSINESS VALUE

This feature makes it easier and quicker to view OCI usage data from within ADB directly. You can then use Database Actions SQL Worksheet, Database Actions Charts, or a data visualization tool, such as Oracle Analytics Cloud, to slice and dice and analyze cost, and usage data.

WORKSHOP: Automatic Partitioning in Autonomous Database

CHALLENGE: Partitioning can help improve the performance of many workloads and make it easier to manage large tables. However, DBAs and non-technical users typically struggle to find the best partitioning method for their data. It requires deep knowledge of the application workload, data schema, and data distribution.

SOLUTION: A partitioning expert in-a-box. ADB's automatic partitioning analyzes your application workload and automatically applies partitioning to your tables and indexes to improve performance or to allow better management of large tables.

Automatic partitioning makes it easier for all users to benefit from partitioning without performing manual schema and workload analysis. Explore how this feature works using this new free workshop on Oracle LiveLabs:

https://bit.ly/adb-auto-p

STRONGER SECURITY
(Shared Infrastructure Only)
Data Safe now runs 100% within the OCI console

CHALLENGE: Customers are looking for a way to integrate their data security processes into the OCI event and notification services. They want to receive notifications and alerts across security topics such as security assessments, data discovery tasks, and masking tasks.

Many cloud admins want to automate their data security processes using scripting to create easy-to-use and repeatable processes.

SOLUTION: Data Safe’s security center is no longer a separate console; it is now 100% within the OCI console framework.

Data Safe can now be integrated with other OCI event-driven services like the OCI notification service. This allows Data Safe alerts to generate notification emails and SMS messages and makes integrating Data Safe with external services like a SIEM or log analyzer easy.

The full complement of Data Safe APIs is now available, allowing easy orchestration of everything from database target registration through data masking to audit report generation and retrieval.

VALUE

These enhancements make using Data Safe even easier for security admins, cloud admins, and fleet admins.

New critical events with customer-managed keys

CHALLENGE: Customers are looking for a way to integrate their use of customer-managed keys into the OCI event and notification services. They want to receive notifications and alerts when there is an issue with the keys.

SOLUTION: Autonomous Database events are triggered when an instance is using customer-managed keys, and the database becomes inaccessible or when the database recovers from being inaccessible. Two new events have been added:

  • DatabaseInaccessibleBegin
  • DatabaseInaccessibleEnd
VALUE

This feature makes it much easier for cloud admins and fleet admins to monitor and get alerts relating to potential issues with the operation of customer-managed keys.

RICHER ANALYTICS
(Shared Infrastructure Only)


Business models tool now integrated into Database Actions Data Analysis tool

CHALLENGE: Many data visualization tools layer a multidimensional model over the underlying relational data model to make it easier for users to slice and dice and analyze their data. However, each tool builds and manages its own model in isolation, making it incredibly difficult to centralize the definition and management of dimensions, hierarchies, attributes, and measures to ensure consistency across reports and charts.

Deriving the multidimensional metadata within each tool takes significant time and effort because it requires a deep understanding of the underlying relational data model.

SOLUTION: The Database Actions Data Analysis tool (replaces the Business Models tool) provides a user-friendly interface to create multidimensional models. The new interface allows users to automatically create Analytic Views from the underlying fact tables and dimension tables in the Autonomous Database.

Additional capability includes new tools to analyze the data using the multidimensional model, gain immediate insights, and review any relevant data quality issues.

The Analytic view created using this tool can be used directly in Oracle Analytics Cloud and many other tools. The curated Analytic View can be exported directly as a Tableau project for further analysis.

VALUE

Metadata can be shared with tools such as Oracle Analytics Cloud, Tableau, and many other tools to create a richer analytical experience
